# 1.RT-2: Vision-Language-Action Models Transfer Web Knowledge to Robotic Control，2023

把两类数据混合训练：
`互联网图文任务：图像 → 文本回答`
`机器人任务：图像 + 指令 → 动作 token`
所以二者都可以使用同一个 Transformer 输出层和交叉熵损失。

这样理解 RT-2：
```markdown
第一层：理解
“哪个是苹果，用户要求做什么”

第二层：预测动作编码
[179,128,102,128,141,128,255]

第三层：动作解码
末端向前 2cm、向下 1cm、夹爪闭合

第四层：传统机器人控制
逆运动学 → 目标关节角 → 电机执行
```

RT-2 的问题：
1.分桶这种方法本身就会存在误差
2.高频连续动作通常变化很平滑，大量相邻 token 几乎完全相同。模型很容易只学会“复制上一步”，而没有真正理解任务。
3.各维度独立分箱，忽略动作结构；传统的“每维度、每时刻分箱”在高频灵巧任务上效果很差。

# 2.π₀: A Vision-Language-Action Flow Model for General Robot Control，2024

π₀保留预训练 VLM 负责理解图像和语言，同时增加一个基于 **flow matching** 的动作专家，生成一整段连续动作：
```
VLM：理解任务和场景
        ↓
Action Expert：从噪声逐渐生成连续动作块
```
它在单臂、双臂和移动操作平台的混合数据上训练，并展示了叠衣服、收拾桌面、组装纸箱等复杂操作。
