## 1.Reward Hacking
### 1.1 InfoRM
$\qquad$这是NeurIPS 2024接收的一篇文章。他们的核心idea是用信息论的角度来约束RM。具体来说，他们引入了变分信息瓶颈（VIB），可以强迫RM在学习的时候过滤掉那些跟人类偏好无关的虚假特征。这个方法有个很实用的副产品：他们发现过度优化跟VIB潜在空间里的outliers（异常点）有关系，基于这个观察提出了一个叫CSI（Cluster Separation Index）的指标，可以用来检测是不是出现了reward hacking。这个指标的好处是可以在线监控，一旦发现苗头不对就能及时干预。

普通优化 = 对 y = f(x) 的 x 求导
**变分 = 对函数求导 / 找最优函数。真实的最优函数是在真实的分布里面，我们用其他的简单分布比如高斯分布近似真实分布，然后从近似分布选一个最优函数出来，这个过程就叫变分推断**。
### 1.2 Reward Shaping
$\qquad$最近有篇工作叫PAR（Preference As Reward）。他们总结了两个设计原则：第一，RL reward应该是有界的，不能无限增长；第二，reward的增长曲线应该是前期快后期慢，逐渐收敛。基于这两个原则，他们提出直接用RM内部的latent preference作为RL signal，而不是用RM的输出分数。
### 1.3 Correlated Proxies
$\qquad$这篇 ICLR 2025 的 Correlated Proxies 把 reward hacking 形式化为：proxy 与 true reward 在 reference policy 覆盖到的 state-action 上相关，但优化 proxy 会把策略推到 reference 很少访问的区域，相关性崩溃并导致真奖励下降。基于此他们证明“优化 proxy + 正则化到 reference”能对真奖励改进给出下界。论文进一步指出：RLHF 常用的 action-distribution KL 正则缺乏这种保证；理论与实证更支持用 occupancy measure 的 χ² 散度做正则。对于 bandit 化的 RLHF，OM 与 AD 等价；但在更长时程环境里 OM 能更好刻画分布漂移，也更能抑制 reward hacking。
- $\chi^2$ 散度：$D_{\chi^2}(P \parallel Q) = \mathbb{E}_Q\left[ \left( \frac{P}{Q} - 1 \right)^2 \right]$，二次惩罚、约束更强（只要 P/Q 稍微偏离 1，一平方，惩罚瞬间爆炸）。 
- KL 散度：$D_{KL}(P \parallel Q) = \mathbb{E}_P\left[ \log \frac{P}{Q} \right]$，对数惩罚、约束更“软”，对分布尾部差异不敏感（log稀释了）。
- Occupancy measure：包括状态占用度量 ρ(s) 和状态-动作占用度量 ρ(s,a)，论文这里指的是ρ(s,a)。**π(a|s) 是局部条件，ρ(s,a) 是全局联合，占用度量 ρ 可以推出动作分布 π，但动作分布 π 推不出 ρ，所以ρ(s,a)信息更多**
>在 LLM/Agent 里，为什么“只看 AD”会漏掉重要变化？
>想象一个工具型 LLM agent（浏览网页 + 写回答），状态 $s$ 里包含：问题、当前草稿、已打开网页内容、搜索结果列表等；动作 $a$ 包括：`Search(q)`、`Open(k)`、`Quote(span)`、继续生成文本 token。 
>**策略 A (reference / safe)**： 
>- 通常先 `Search("官方文档 + 指标定义")` $\to$ `Open(官方站)` $\to$ `Quote` $\to$ 写答案 
>**策略 B (hacked)**： 
>- 只把第一步改一点点：更偏向 `Search("... 速成/模板")` 或 `Open(SEO 博客)` 
>- 后面在“读到网页内容后怎么继续写”的 $\pi(a|s)$ 可能和 A 很像（都在继续生成、引用、总结） 
>
>这时你会看到： 
>- 在某些“相同页面/相同上下文状态”上，B 的 AD（$\pi(a|s)$）跟 A 差不多； 
>- 但 B 更常走到“低质量网页/噪声页面/错误定义”的状态桶里（也就是 $d_\pi(s)$ 变了），导致整条任务分布被带偏。 
>
>这正是论文强调 OM 的动机：**AD 只约束“在一个状态怎么动”，OM 还约束“你会不会走到那些危险/OOD 状态”；而 reward hacking 往往就是“走到 safe policy 几乎不去的状态”**。 
>
>那为什么你会感觉 “OM 就是 AD 拼起来”？ 
>因为在当前很多 LLM RLHF 的范式里，有一组“很强的假设”使两者变得等价/接近（比如把问题当 contextual bandit：状态分布主要由数据集 prompt 给定，而不是由策略一步步把自己带到新状态）。这篇论文自己也在文中指出：在现行 LLM RLHF 实现下，AD regularization 和 OM regularization 在严格假设下是等价的。 
> **一句话回答：** OM 当然包含 AD（因为 $\rho(s,a) = d(s)\pi(a|s)$），但它还包含策略诱导的状态访问分布 $d_\pi(s)$。在长时程/agentic 场景里，$d_\pi(s)$ 往往才是“出事”的地方，所以 OM 不等于“只有 AD”。

Reward Hacking这个问题本质上反映了我们在做alignment时的一个根本困境：我们很难完美地把真实目标形式化成一个可优化的函数，这个领域在快速发展。
