cmd运行docker info显示的代理还是没有变的：

- Windows 本地代理（如 `127.0.0.1`）在 WSL 内部无法直接访问（WSL 有独立的网络命名空间）。
- Docker Desktop 通过 `http.docker.internal` 将 Windows 代理转换为 WSL 内部可访问的地址，实现跨环境的代理通信。
